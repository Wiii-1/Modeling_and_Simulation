# Output Analysis of a Single-Server Queue Simulation

## Introduction

This is a single-server queue simulation. Customers arrive over time, wait if the server is busy, get served one at a time, then leave the system. The goal of this study is to analyze the output of the simulation and estimate the long-run or steady-state behavior of the queue. In simulation output analysis, the data generated by the simulation run is called the trace, and this output is used to evaluate measures such as the mean and variability of system performance. 

This simulation is treated as **non-terminating** because the focus is not on a fixed ending time. Instead, the goal is to study how the system behaves in the long run after the startup or warm-up period has passed. A non-terminating simulation begins with a transient state and gradually moves toward steady-state, so output analysis is needed to avoid bias from the early observations. 

## Model Description

The model represents a queue with one server and many customers. Each customer arrives at a random time, and each one also has a random service time. If the server is free, service starts immediately. If the server is busy, the customer waits in line until the previous customer is finished.

The main performance measure used in this study is the **waiting time** of each customer. This is a good measure because it shows if the system is operating efficiently or if the queue is becoming congested. The simulation also records arrival time, service start time, departure time, time in system, and queue length on arrival.

## Method Used

Three methods were used in this output analysis: Welch Method, Replication-Deletion, and Batch Means. These methods are all appropriate for non-terminating simulations because they help handle warm-up bias and estimate steady-state performance. 

The Welch Method was used first to identify the warm-up period. This is important because the simulation starts with an empty system, so the early waiting times are usually lower than normal. Replication-Deletion was then used by running several replications, deleting the warm-up observations, and computing the average waiting time from the remaining data. Batch Means was also used by taking one long run, removing the warm-up portion, dividing the rest into batches, and averaging each batch. 

## Results

The simulation output showed that the system begins with startup bias because the queue starts empty. This means the first observations are not enough to represent the normal long-run behavior of the system. Because of that, the warm-up period had to be identified and removed before making the final estimate. 

After removing the warm-up portion, the remaining output was analyzed using Replication-Deletion and Batch Means. These methods were used to estimate the steady-state average waiting time of customers in the queue. The results from these methods provide a more realistic measure of system performance than simply averaging all data from the very start of the run. 

## Discussion

The main point of the output analysis is that not all simulation output should be trusted immediately. Since the model starts in an empty and unrealistic condition, the early waiting times are biased and usually lower than what would happen during normal operation. This is why warm-up bias must be considered in non-terminating simulation studies. 

Using all three methods made the analysis stronger. Welch helped identify where the transient period ends, Replication-Deletion gave a steady-state estimate from multiple independent runs, and Batch Means gave another estimate from one long run. If the results from the final methods are close to each other, that suggests the steady-state estimate is reasonable.

## Conclusions

This study shows that the single-server queue is a non-terminating simulation because it is analyzed for long-run behavior instead of a fixed stopping condition. Since the simulation starts with a warm-up period, output analysis is needed before making conclusions about system performance. 

Welch Method, Replication-Deletion, and Batch Means were suitable for this simulation because they help reduce warm-up bias and improve the steady-state estimate. Overall, the simulation output shows that the queue should be evaluated using post-warm-up data, not raw startup data, in order to get a more reliable result.

Basically, the output analysis shows that the single-server system takes a long time to serve all customers in the queue because customers keep piling up while service is limited. This limited service causes waiting time to increase, especially during peak hours.
